#### 4. Text Analytics
NLP for text mining and analysis
   1. Sentiment analysis : apply labels and confidence score to text at sentence and document level
   2. Opinion mining : when provided with more granular data with a subject and opinion tied to a seniment
   3. Key phrase extraction : identify main concepts in text, works best with larger amounts of text
   4. Language detection
   5. Named Entity Recognition : detect words and phrases mentioned in unstructured data that can be associated with a semantic type ie: location, age, events, person, diagnosis
      
#### 5. OCR Computer Vision
- Extracting text  into a digital format
Azure has 2 APIs:
   1. OCR API - older, only images, executes synchronously, better for less text, more languages,easier to implement
   2. Read API - newer model, images and PDFs, executes asynchronously,  more text, fewer languages, difficult to implement

OCR uses computer vision SDK

#### 6. Form Recognizer  
- Specialized OCR service that preserves structure and relationship of form-like data
- It can identify : key-value pairs, selection marks, table structures
- Outputs: original file relationships, bounding box, confidence scores
Composed of : 
   - Custom models - trained with your own data. Two learning options: train without labels, train with labels
   - Pre-built models for invoices, receipts, business cards, IDs
   - The Layout models
#### 7. LUIS
Language Understanding - no-code ML service to build natural language into apps
Composed of a schema that is autogenerated which defines:
- Intentions - WHat user is asking for: Has NONE intention
- Entities - What parts of the intent is used to determine the answer
- Utterances - examples of user input that includes intent & entities to train ML model match predictions against real user input
  
#### 8. QnA Maker
- Enables conversational layer over your data based on knowledge base
  
  **Features**:
- Knowledge base : Imports content into a knowledge base of Question-Answer pairs. Stores text in mArkdown
- Chatbox : platform to converse with bot
- Chitchat : allows addition of a pre-populated set of the top chit-chat into knowledge based. Has about 100
- Layered Ranking System: data is stored in azure search(1st ranking layer), top results are passed into QnA Maker NLP re-ranking model to produce the final results and confidence score
- Multi-turn conversation : follow -up prompts and context to manage the multiple turns from one question to another
  
**Use cases:**
When you 
- Have Static Info : 
- Provide the same answer to a repeat request 
- Want to filter static info based on meta-information
- Manage a bot conversation that includes static info

### AZURE AI FOUNDRY(formerly Azure AI Studio)   
- PaaS for customizing language models 
- Collection of tools, services to build, manage  and use AI models, appications and agents at scale

**Core capabilities**
1. Foundry Models : model catalog that is a central hub to discover, compare and deploy models ~1800 models in catalog
   - Access to models ie : Azure OpenAI, 3rd party (Anthropic), Open Source(LLama)
   - Deploy models as agents
   - Manage lifecycle using version control
       
2. Agent Service :
   - Playground with ready-to-use environment for testing ideas
   - Low code or code-first development of AI agents, built in integrations
     
3. Foundry Tools :
   - Build, test, demo and deploy tools for gen AI
   - Prebuilt azure services for Vision, Language, Speech, Document Intelligence, Content Safety
   - Accessible via: SDKs, APIs, Portal
   - Prompts 
   - Fine tuning : further train a pretrained model on a smaller, domain-specific dataset for better accuracy; updates weights and biases within the model; change             behaviour of model where system prompt is not sufficient.
   - Distillation : when another LLM creates training data for fine tuning an SML   
  
4. Governance and Observability
   - Unified dashboard for performance metrics, safety checks, lifecycle monitoring
   - Evaluation - test for groundedness
   - Tracing - understand the flow, interactions between components
   - Safety + Security 
   - Continuous feedback loops
   - Ensures responsible AI through compliance, risk mitigation, identity management

### 4. COMPUTER VISION IN AZURE & AI FOUNDRY
Computer vision services:
1. Image Analysis services : object detection, tagging, generate captions, supports OCR
2. Azure AI Face service : detect,recognize and analyze human faces ; provides models for advanced facial analysis


**Azure Vision Image Analysis : Capabilities**
Prebuilt:
1. Image Captioning : generate human readable captions for images
2. Object detection : identify common objects with confidence scores, returns the bounding box coordinates 
3. Tagging : suggest tags for indexing and searching
4. OCR : extract text from images

Custom Model Training:
- Uses transfer learning hence you can use complex models with few images
1. Train image classification model to predict category of an image
2. Object detection model to detect and classify multiple objects with bounding boxes


**Azure Vision Face Service : Capabilities**
Core concepts:
1. Face detection - identifies regions in an image that contain human face ; returns bounding box for each face; detects facial landmarks
2. Face recognition - trains models to identify known individuals from facial features ; supports identity verification and access control

**Face Service features**
- Accessories - detects headware, glasses, masks (with confidence score)
- Blur - clarity of the face
- Exposure - check if face is over/under exposed
- Head pose - face orientation in 3d
- Mask - detect mask usage
- Noise - measure image graininess
- Occlusion - if objects block the face
- Quality for recogniton - rate image quality for recognition
